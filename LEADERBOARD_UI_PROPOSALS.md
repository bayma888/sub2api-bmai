# 排行榜 UI 方案对比

> 创建日期：2026-04-01
> 关联文档：[LEADERBOARD_FRONTEND_PLAN.md](./LEADERBOARD_FRONTEND_PLAN.md)、[LEADERBOARD_GAMIFICATION_PLAN.md](./LEADERBOARD_GAMIFICATION_PLAN.md)
> 状态：方案C 已选定

## 背景

原版排行榜使用"领奖台 Top3 + 列表 4-20"的布局（见 LEADERBOARD_FRONTEND_PLAN.md）。
实际运行后发现以下问题：

- 用户少时领奖台很空旷，只有1个人的领奖台尴尬
- 竖向领奖台 + 列表的信息密度不高，视觉冲击力不够
- 用户需要切Tab才能看不同榜单，80%用户只看默认第一个

于是提出以下4个改进方案。

### 段位计算逻辑（草案）

```typescript
function getTier(rank: number, total: number, value: number): Tier {
  if (rank === 1) return 'champion'  // Top 1 = 王者
  if (rank <= 3) return 'diamond'    // Top 3 = 钻石
  if (rank <= total * 0.1) return 'platinum'  // Top 10% = 白金
  if (rank <= total * 0.3) return 'gold'      // Top 30% = 黄金
  if (rank <= total * 0.6) return 'silver'    // Top 60% = 白银
  return 'bronze'  // 其余 = 青铜
}
```
